# 📖 Documentation des Composants Blog

Cette documentation t'explique comment utiliser chaque composant pour créer des articles de blog facilement et rapidement.

## 🚀 Installation et Setup

1. **Copie le fichier des composants** dans `components/blog/blog-components.tsx`
2. **Importe les composants** dont tu as besoin dans ton article
3. **Utilise le template complet** comme base et modifie selon tes besoins

## 📋 Structure d'un Article Type

```typescript
// Imports essentiels
import { Metadata } from "next";
import { generateSEOMetadata } from "@/lib/seo";
import { StructuredData } from "@/components/seo/structured-data";
import { Breadcrumb } from "@/components/ui/breadcrumb";

// Imports des composants blog
import {
  BlogHeader,
  KeyPoints,
  ProductCard,
  // ... autres composants
} from "@/components/blog/blog-components";

// Imports des sections reset existantes
import { SpirulineResetSection } from "@/components/sections/spiruline-reset-section";
```

## 🎯 Guide Composant par Composant

### 1. BlogHeader - En-tête d'article

**Usage :**
```tsx
<BlogHeader
  title="Titre Principal de l'Article"
  excerpt="Description courte et accrocheuse de l'article"
  badges={[
    { text: "Nouveau", variant: "default" },
    { text: "Guide", variant: "secondary" }
  ]}
  author="Thibaut"
  date="30 janvier 2025"
  readingTime="5 min de lecture"
/>
```

**Props :**
- `title` (string, requis) : Titre principal
- `excerpt` (string, requis) : Description/sous-titre
- `badges` (array, optionnel) : Badges avec texte et variante
- `author` (string, optionnel) : Nom de l'auteur (défaut: "Thibaut")
- `date` (string, optionnel) : Date de publication
- `readingTime` (string, optionnel) : Temps de lecture estimé

### 2. KeyPoints - Points clés/résumé

**Usage :**
```tsx
<KeyPoints
  title="🌟 Ce que vous allez découvrir :"
  points={[
    "Premier point important",
    "Deuxième bénéfice",
    "Troisième avantage"
  ]}
  variant="success"
/>
```

**Props :**
- `title` (string, optionnel) : Titre de la section
- `points` (string[], requis) : Liste des points clés
- `variant` ("success" | "info" | "warning", optionnel) : Couleur du thème

### 3. ProductCard - Carte produit avec prix

**Usage :**
```tsx
<ProductCard
  name="Spiruline Premium - 80 comprimés"
  description="2g de spiruline pure par dose"
  price="37€"
  dosage="4 comprimés par jour"
  ctaText="Commander maintenant"
  ctaLink="https://lien-vers-produit.com"
  highlight={true}
/>
```

**Props :**
- `name` (string, requis) : Nom du produit
- `description` (string, requis) : Description courte
- `price` (string, requis) : Prix affiché
- `dosage` (string, requis) : Posologie
- `ctaText` (string, optionnel) : Texte du bouton
- `ctaLink` (string, requis) : Lien vers le produit
- `highlight` (boolean, optionnel) : Mise en surbrillance

### 4. BenefitsGrid - Grille de bienfaits avec icônes

**Usage :**
```tsx
<BenefitsGrid
  title="Bienfaits Principaux"
  benefits={[
    {
      icon: BlogIcons.Energy,
      title: "Énergie naturelle",
      description: "Augmente naturellement votre niveau d'énergie"
    },
    {
      icon: BlogIcons.Shield,
      title: "Immunité renforcée",
      description: "Soutient votre système immunitaire"
    }
  ]}
  columns={2}
/>
```

**Props :**
- `title` (string, optionnel) : Titre de la section
- `benefits` (Benefit[], requis) : Liste des bienfaits avec icône, titre et description
- `columns` (1 | 2 | 3, optionnel) : Nombre de colonnes

**Icônes disponibles :**
```typescript
BlogIcons.Energy    // ⚡ Énergie
BlogIcons.Shield    // 🛡️ Protection/Immunité
BlogIcons.Brain     // 🧠 Cerveau/Cognition
BlogIcons.Heart     // ❤️ Cœur/Cardiovasculaire
BlogIcons.Eye       // 👁️ Vision
BlogIcons.Muscle    // 💪 Muscle/Force
BlogIcons.Star      // ⭐ Excellence
BlogIcons.Award     // 🏆 Récompense
BlogIcons.Target    // 🎯 Objectif
BlogIcons.Clock     // ⏰ Temps
BlogIcons.Users     // 👥 Communauté
BlogIcons.Lightbulb // 💡 Idée/Conseil
```

### 5. Testimonials - Témoignages clients

**Usage :**
```tsx
<Testimonials
  title="Ce que disent nos clients"
  testimonials={[
    {
      name: "Marie",
      rating: 5,
      text: "Excellent produit, je le recommande !",
      verified: true,
      location: "Paris"
    }
  ]}
  averageRating={4.8}
  totalReviews={2847}
/>
```

**Props :**
- `title` (string, optionnel) : Titre de la section
- `testimonials` (Testimonial[], requis) : Liste des témoignages
- `averageRating` (number, optionnel) : Note moyenne
- `totalReviews` (number, optionnel) : Nombre total d'avis

### 6. NutritionalTable - Tableau nutritionnel

**Usage :**
```tsx
<NutritionalTable
  title="Composition et valeurs nutritionnelles"
  subtitle="Pour 4 comprimés (dose quotidienne)"
  values={[
    { nutrient: "Protéines", amount: "1,3 g", daily: "" },
    { nutrient: "Fer", amount: "2,4 mg", daily: "(17% VNR*)" }
  ]}
  footer="*VNR : Valeurs Nutritionnelles de Référence"
/>
```

### 7. UsageGuide - Guide d'utilisation étapes

**Usage :**
```tsx
<UsageGuide
  title="Comment utiliser ce produit ?"
  steps={[
    {
      title: "Dosage standard",
      description: "4 comprimés par jour avec un verre d'eau"
    }
  ]}
  tips={[
    "Commencez par une demi-dose",
    "Prenez avec de la vitamine C"
  ]}
/>
```

### 8. AlertBox - Alertes et mises en garde

**Usage :**
```tsx
<AlertBox type="warning" title="⚠️ Précautions importantes">
  <p>Contenu de l'alerte avec du HTML possible</p>
  <ul>
    <li>Point 1</li>
    <li>Point 2</li>
  </ul>
</AlertBox>
```

**Types disponibles :**
- `"warning"` : Jaune, pour les avertissements
- `"danger"` : Rouge, pour les dangers
- `"info"` : Bleu, pour les informations
- `"success"` : Vert, pour les confirmations

### 9. BeforeAfter - Comparaison avant/après

**Usage :**
```tsx
<BeforeAfter
  title="Résultats après 3 mois"
  before={{
    title: "Avant",
    items: ["Fatigue constante", "Immunité faible"]
  }}
  after={{
    title: "Après",
    items: ["Énergie stable", "Résistance accrue"]
  }}
/>
```

### 10. ContentSection - Section contenu avec image

**Usage :**
```tsx
<ContentSection
  title="Titre de la section"
  imageSrc="/chemin/vers/image.jpg"
  imageAlt="Description de l'image"
  imagePosition="right"
  content={
    <div>
      <p>Contenu de la section...</p>
    </div>
  }
  ctaButton={{
    text: "En savoir plus",
    link: "/lien",
    variant: "outline"
  }}
/>
```

### 11. FinalCTA - Call-to-action final

**Usage :**
```tsx
<FinalCTA
  title="🌟 Prêt à commencer ?"
  subtitle="Rejoignez des milliers de personnes satisfaites"
  ctaText="🛒 Commander maintenant"
  ctaLink="https://lien-commande.com"
  backgroundColor="bg-gradient-to-br from-green-600 to-blue-600"
  responsive={true}
/>
```

## 📝 Workflow de Création d'Article

### Méthode 1 : À partir du template

1. **Copie le template complet** (`template-article-complet/page.tsx`)
2. **Renomme le fichier** selon ton article
3. **Modifie les métadonnées SEO** (titre, description, mots-clés)
4. **Remplace le contenu** section par section
5. **Supprime** les composants dont tu n'as pas besoin
6. **Ajuste** les liens et images

### Méthode 2 : Composant par composant

1. **Crée un nouveau fichier** article
2. **Importe les composants** nécessaires
3. **Assemble** selon tes besoins
4. **Teste** le rendu
5. **Publie**

## 🔧 Customisation Avancée

### Couleurs personnalisées

Tu peux modifier les couleurs de n'importe quel composant :

```tsx
<KeyPoints variant="info" />  // Bleu
<KeyPoints variant="warning" />  // Jaune  
<KeyPoints variant="success" />  // Vert (défaut)
```

### Responsive personnalisé

Les composants sont responsive par défaut, mais tu peux ajuster :

```tsx
<BenefitsGrid columns={3} />  // 3 colonnes sur desktop
<BenefitsGrid columns={1} />  // 1 colonne partout
```

## 🚨 Bonnes Pratiques

### SEO
- **Toujours inclure** les métadonnées avec `generateSEOMetadata()`
- **Utiliser** les structured data avec `<StructuredData />`
- **Ajouter** les breadcrumbs avec `<Breadcrumb />`

### Performance
- **Optimiser les images** (format WebP, tailles appropriées)
- **Lazy load** les images non critiques
- **Tester** sur mobile

### Accessibilité
- **Alt text** pour toutes les images
- **Hiérarchie** des titres respectée (H1 > H2 > H3...)
- **Contraste** suffisant pour les textes
